# ROADMAP.md — Forge Local Runtime

## Phase 0 — project establishment and doctrine

Deliver:

- repo skeleton
- top-level constitutional docs
- decision records
- service map
- boundary matrix
- runtime denial and degraded vocabulary

Exit condition:

The project has stable architectural authority before implementation expands.

## Phase 1 — DF Local Foundation alignment

Deliver:

- DF Local Foundation role document
- migration lifecycle model
- backup/restore/export boundaries
- app registration model
- readiness model
- degraded and unavailable schema posture

Exit condition:

The local data/control substrate is clearly bounded and does not drift into app logic.

## Phase 2 — NeuronForge Local alignment

Deliver:

- task/inference boundary alignment
- candidate-not-authority rule integration
- trust-surface alignment
- degraded inference reporting model

Exit condition:

NeuronForge Local is formally aligned as bounded local inference substrate.

## Phase 3 — Cortex alignment

Deliver:

- file-intelligence boundary alignment
- syntax-versus-semantics rule
- retrieval-preparation and handoff contract alignment
- privacy-preserving visibility defaults
- observation-boundary doctrine

Exit condition:

Cortex is clearly defined as service-only local file intelligence boundary.

## Phase 4 — FA Local alignment

Deliver:

- execution boundary alignment
- policy-gated routing rules
- requester-trust and capability-admission model
- guarded execution model
- execution state/reporting alignment
- forensic minimization doctrine

Exit condition:

FA Local is clearly defined as governed local execution boundary.

## Phase 5 — cross-service handoff and anti-drift review

Deliver:

- cross-service handoff contracts
- readiness and status integration rules
- anti-drift audit across all four services
- split-trigger review
- runtime observability/privacy audit

Exit condition:

The local runtime layer remains coherent, bounded, privacy-preserving, and non-authoritative.

## Sequencing rule

No implementation expansion should outrun:

- ownership artifacts
- shared schema posture
- denial/degraded vocabulary
- boundary matrix
- runtime doctrine
